BEFORE WE BEGIN
WHY WE CAN'T HOST THIS ONLINE
🦀
Native Rust Desktop Application
KitsuneEngine is a compiled Rust binary — not a web app. It embeds a real OS-level WebView (WebView2 on Windows, WebKit on macOS) that renders actual web pages. Cloud servers have no display, no GPU, no OS WebView stack to render against.
🖥️
Requires Display Server & GPU Access
The browser engine relies on OS-level compositing, hardware GPU acceleration, and a windowing system (Win32 / Cocoa). Render.com and similar PaaS platforms run headless Linux containers — no display, no windowing, no WebView.
🔐
Vault & Credential Security Requires Local Trust
The Argon2id encrypted vault and session tokens are bound to the local machine's environment. Running in a shared cloud container would expose the trust boundary the entire security model is built on — defeating the purpose.
⚡
AMD ROCm Inference Is Device-Local by Design
The whole point of AMD integration is that sensitive agent reasoning never leaves the user's hardware. Remote hosting would eliminate the on-device privacy guarantee that differentiates Kitsune from every cloud-based competitor.

THE PROBLEM
AI AGENTS ARE BLIND TO THE WEB
01
Screenshot Scraping
Agents see static JPEGs, not live DOM. Miss dynamic elements, SPA transitions, invisible state. A blind agent guessing at a webpage.
02
CDP / DevTools Hacks
Fragile, detectable, browser-version-locked. Breaks with every Chrome update. Not production-grade; barely proof-of-concept.
03
Extension API Sandbox
Extensions can't touch real sessions, cookies, or auth flows. They're walled off by design — and AI agents are further boxed in inside that box.
04
Zero Safety Layer
Any agent action — payments, emails, account creation — fires without oversight. One hallucination = irreversible real-world consequence.

THE SOLUTION
KITSUNE — COMPLETE AGENT SOVEREIGNTY
Claude Code changed how developers work — because agents got real terminal access. Kitsune does this for the browser. Built in Rust. Secured by the compiler. Privacy is the architecture that makes full control safe to hand over.
01
Full DOM Read & Write
Live DOM access — not screenshots. Understands page structure, SPA transitions, dynamic elements — then modifies or triggers any of it.
02
Vault-Encrypted Sessions
Argon2id + Age encryption. Credentials stored as opaque tokens — agents receive handles, never plaintext. Zero exposure by construction.
03
Human-in-the-Loop Gates
Payments, accounts, emails — blocked until explicit human approval. Non-cloneable, single-use, 30s TTL. Type-system enforced — bypass impossible.
04
Multi-Tab Orchestration
Agents open, manage, and coordinate across tabs simultaneously — like a developer with 12 windows, never confused. Parallel execution natively.
05
No-Code Agent Builder
Describe intent in plain English. Converted to repeatable, shareable, auditable agent specs — no selectors, no scripting, no boilerplate.
06
Process-Level Sandboxing
Each tab is an isolated OS process. Rust ownership makes cross-tab memory access a compile-time error — not a runtime check. Memory safe by design.

AMD INTEGRATION
ON-DEVICE INTELLIGENCE. HARDWARE-SECURED.
🔴 Local LLM via Ollama + ROCm
Kitsune's AiRouter directs sensitive task types — VaultDecision, SensitiveForm — to always-local inference. AMD GPUs run Ollama models at full throughput. Agent reasoning never leaves the device. This is structural, not configurable.
⚡ Data never leaves hardware
🔴 Multi-Agent Swarm Compute
Each swarm agent runs a browser context and an LLM inference call in parallel. AMD MI300X unified memory handles the working-set of 10+ simultaneous agent contexts with the throughput needed for real-time orchestration.
⚡ Parallel agent throughput
🔴 Roadmap: candle + ROCm
kitsune-ai LocalAiBackend is feature-gated and scaffolded for the candle Rust ML framework + AMD ROCm. This will enable in-process, zero-copy model serving — no IPC overhead, no Ollama dependency, pure Rust inference on AMD silicon.
⚡ Zero-IPC local inference
🔐 Air-Gapped Enterprise
AMD ROCm enables fully offline enterprise deployments. Sensitive healthcare, legal, and financial data never leaves the premises. Compliance by hardware architecture.
